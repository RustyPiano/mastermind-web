# 密码机 进阶体验与社交扩展计划

**目标**：提升游戏沉浸感，增加零后端成本的社交裂变传播能力，并用成就与评价系统丰富用户的长线游玩动机。

**核心聚焦**：
1. 视觉动效升级 (Animations)
2. 异步社交对战 (Asynchronous UI/URL PvP)
3. 结算评价系统 (Coach Mode / Analytics)
4. 徽章成就系统 (Achievements)

---

## 📅 任务拆解与执行状态

### [x] Task 1: 视觉动效升级 (Animations & Feedback)
**目标:** 增强“开奖悬念”和误操作的物理反馈。
- **[x] 1.1 错误提示动效：** 在未选满颜色即点击“提交猜测”时，给当前活动行或状态提示框添加左右抖动 (Shake) 的 CSS 动画。
- **[x] 1.2 逐个揭晓动效：** 提交猜测后，反馈点（🟢🟠⚪）不再瞬间全部出现，而是通过 CSS `animation-delay` 或 JS `setTimeout` 实现依次弹出的交错效果（如间隔 150ms），营造悬念。
- **[x] 1.3 胜利/失败表现：** 失败时棋盘整体压暗，重点照亮顶部的正确答案；胜利时让答案行的颜色球产生轻微的波浪跳跃动画。

### [x] Task 2: 纯前端异步社交对战 (Async PvP via URL)
**目标:** 让双人对战不再局限于“同设备轮流”，而是能生成专属链接发给朋友。
- **[x] 2.1 生成挑战链接：** 在“设置密码”完成界面，增加一个“生成挑战链接”按钮。将设定的秘密配置（如秘密颜色数组）使用 Base64 进行轻量混淆，拼入 URL（例如 `?challenge=eyJzZWNyZXQi...`）。
- **[x] 2.2 启动路由解析：** 在 `js/game.js` 初始化时检查 URL 参数。若存在 `challenge`，解析数据并强制进入一个特殊的“好友挑战模式”，跳过随机生成环节直接锁定答案。
- **[x] 2.3 挑战专属结算与分享：** 通关此模式后，结算文案变为“你破解了朋友留下的难题！”，且“分享结果”生成的文本里要包含该挑战链接，方便朋友再接力传给其他人。

### [x] Task 3: 结算评价系统 (Coach Mode / Analytics)
**目标:** 不再只有冰冷的“赢/输”，而是根据玩家表现给出“教练式的评价”。
- **[x] 3.1 评价逻辑计算：** 根据所选模式的难度和玩家实际使用的步数计算评级（S、A、B、C）。
  - 例如：`<=3` 步为 S级（福尔摩斯！）；`<=5` 步为 A级（思路清晰）；`<=8` 步为 B级（稳扎稳打）；只剩最后机会过关为 C级（惊险绝杀）。
- **[x] 3.2 UI 整合：** 将此评价文字（配合简短的一句鼓励/夸赞）展示在游戏结束的弹窗 (`overlay-card`) 中。

### [x] Task 4: 本地成就与徽章系统 (Achievements)
**目标:** 给用户提供长线游玩收集目标。
- **[x] 4.1 定义成就库：**
  - “一发入魂”：第1次猜测直接通关。
  - “极限绝杀”：用掉最后一次机会才通关。
  - “盲人摸象”：第一步猜测获得全 ⚪ (完美避开所有正确答案)。
- **[x] 4.2 数据存储扩展：** 更新 `js/stats.js` 和 `js/storage.js` 的 schema，增加 `achievements: []` 字段，并在每局结束时自动判定新解锁的成就并存入 `localStorage`。
- **[x] 4.3 徽章墙 UI (精简版)：** 在主页的“详细统计面板”中，增加一个简单的成就陈列区，用 emoji（如 🏆🥇🎯）表示已解锁的成就，灰色或 `?` 表示未解锁。

## 🧪 测试验证计划

1. **Task 1 相关验证:** 快速填入颜色并提交，确认反馈点是“逐个”出来的，而不是“一起”出来的。确认未填满时按动按钮有抖动。
2. **Task 2 相关验证:** 自己创建一局“双人对战”，生成链接后，在新页签或手机端打开该链接，检查是否自动锁定了设定的答案并直接进入猜测环节。完成游戏后，检查分享文案。
3. **Task 3 相关验证:** 在入门模式下，伪造一个 3 步通过的数据，验证是否显示 S级文案。伪造一个 10/10 惊险通关，验证是否显示 C级文案。
4. **Task 4 相关验证:** 故意在第一步乱填几个保证出错的颜色，触发“盲人摸象”，然后游戏结束后回到主页面板，确认新增了该成就标识。
